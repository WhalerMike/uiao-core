# machine/schemas/

JSON Schema, YAML Schema, OSCAL profiles, and OpenAPI definitions.

## Rules

- Only machine-parseable formats (`.json`, `.yaml`, `.yml`, `.xml`, `.oscal`).
- No Markdown or prose.
- Schemas must use semantic versioning in filenames (e.g., `schema.adapter.v1.2.0.json`).
- Changes require a corresponding update in `docs/architecture/` or `docs/governance/`.
