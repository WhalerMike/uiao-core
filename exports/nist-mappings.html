NIST 800-53 Rev. 5 Control Mappings

This document provides a high-fidelity mapping of the uiao-core architecture planes to the NIST 800-53 Rev. 5 control families. These mappings are critical for the agency’s FedRAMP 20x Phase 2 Pilot (concluding March 31, 2026) and establish the compliance baseline for the transition to Phase 3 in Q3 2026.

1. Identity Plane (Zero-Trust Core)

Focus: Access Control (AC) and Identification and Authentication (IA)

Control ID Control Name uiao-core Implementation
AC-2 Account Management Centralized Azure AD integration for all network administrative and user accounts.
AC-3 Access Enforcement ISE-driven SGT assignment ensures privilege-based access at the network ingress.
IA-2 Identification and Authentication (Org Users) Enforces MFA and certificate-based authentication (EAP-TLS) via the Identity Plane.
IA-8 Identification and Authentication (Non-Org Users) Automated Guest-access workflows via ISE, providing strict isolation for contractor/external entities.

2. Addressing Plane (Visibility and Inventory)

Focus: Configuration Management (CM) and System and Comm. Protection (SC)

Control ID Control Name uiao-core Implementation
CM-2 Baseline Configuration appendices.yml acts as the immutable baseline for all site IP and DNS configurations.
CM-8 System Component Inventory Automated Infoblox IPAM provides a real-time, authoritative inventory of every connected endpoint.
SC-20 Secure Name / Address Resolution DNSSEC and automated DNS zone protection (Appendix A-04) ensure the integrity of the name resolution plane.
SC-21 Secure Name / Address Resolution (Recursive) Centralized DNS hierarchy prevents DNS poisoning and ensures recursive queries follow agency-approved paths.

3. Overlay Plane (Segmentation and Isolation)

Focus: System and Communications Protection (SC)

Control ID Control Name uiao-core Implementation
SC-7 Boundary Protection Overlay fabric utilizes VRFs (Macro-segmentation) to create immutable boundaries between security zones.
SC-39 Process Isolation / Segmenting SGACLs (Appendix BA-02) enforce micro-segmentation, preventing lateral movement between disparate host roles.
SC-8 Transmission Confidentiality and Integrity SD-WAN Edge (Appendix AA-02) mandates AES-256-GCM encryption for all inter-site data in transit.

4. Telemetry Plane (Accountability and Monitoring)

Focus: Audit and Accountability (AU) and System and Info Integrity (SI)

Control ID Control Name uiao-core Implementation
AU-2 Event Logging Automated Splunk integration captures every CLI change and automated API call in the uiao-core ecosystem.
AU-6 Audit Record Review, Analysis, and Reporting Closed-loop feedback system correlates identity (SGT) with traffic logs for rapid incident reconstruction.
SI-4 System Monitoring Real-time telemetry via Azure Monitor provides continuous monitoring of infrastructure health and security posture.

FedRAMP 20x Phase 2 Compliance Note

As of March 21, 2026, these mappings have been validated against the current Pilot requirements. Full operationalization of these controls is a prerequisite for the Q3 2026 transition to FedRAMP 20x Phase 3.
